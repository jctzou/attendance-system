# 打卡系統設計規格書 (Attendance Clock-in Spec)

> **版本**: v1.3
> **狀態**: 已實作 (Implemented)
> **目標**: 建立直覺、防呆的員工打卡介面，並區分月薪與鐘點制的操作體驗。

---

## 1. 頁面功能概觀與角色權限

本頁面為所有 **員工 (Employee/Manager)** 每日使用的核心功能。
負責即時記錄上下班時間，並立即回饋當天的出勤狀態（正常/遲到/早退）。

---

## 2. 介面佈局與資料流 (UI Layout & Data Flow)

### 2.1 卡片佈局 (Card System)
介面以「現代化卡片 (Modern Card)」為核心，位於畫面正中央，具有強烈陰影 (`shadow-[0_20px_50px_rgba(0,0,0,0.05)]`) 與圓角。
- **時鐘區**: 顯示伺服器與客戶端同步的實時標準時間，採用超大字級 (`text-6xl`) 確保清晰。
- **操作區**: 依據班別與當前狀態，動態渲染上班或下班按鈕。
- **狀態回饋區**: 打卡完成後立即顯示標籤 (e.g., 綠色 Normal / 橘色 Late / 紅色 Early Leave) 與上下班確切時間。

### 2.2 鐘點制與月薪制的 UI 差異
- **月薪制 (Monthly)**:
    - 顯示規定上下班時間 (如 09:00 - 18:00)。
    - 打卡僅為 **單一按鈕**，無需另外選擇時間。
- **鐘點制 (Hourly)**:
    - 隱藏規定上下班時間。異常狀態標籤（正常/遲到/早退）預設隱藏，受全域變數 `NEXT_PUBLIC_SHOW_HOURLY_STATUS` 控制。
    - 配合全域校正政策，**不提供**原生打卡按鈕，改為渲染「時間段選擇器 (`TimeSlotSelector`)」，動態顯示現在最接近的兩個 30 分鐘級距按鈕（例：現在 09:15，則出現 09:00 與 09:30 選項）。
    - 下班打卡時，額外提供午休時間的下拉選單 (1h / 1.5h / 2h 等) 供扣除淨工時使用。

## 3. 特殊頁面邏輯與狀態管理 (Local Logic & State)

### 3.1 樂觀更新與即時回饋 (Optimistic UI)
- Server Action 送出後，前端必須**立即**利用 React 狀態更新 UI 至「上班中」或「已下班」，並顯示狀態標籤與工作時數。
- **禁止** 等待頁面 Reload 或整頁重新抓取資料才顯示結果，以消除點擊延遲感。

### 3.2 誤按防呆：取消下班 (Cancel Clock Out)
- 當日若已有 `clock_out_time`，介面應提供「取消下班」的微型按鈕。
- 點擊後，必須彈出系統標準的 `ConfirmDialog` 再次確認。確認後才清除下班時間並回復為上班中。

### 3.3 重複打卡防護
- 點擊打卡瞬間，立刻鎖定按鍵 (`isLoading = true`)。
- 若同時跨裝置送出導致重複，Server Action 會回傳 `DUPLICATE_ENTRY`，前端顯示提示訊息不再疊加新紀錄。
