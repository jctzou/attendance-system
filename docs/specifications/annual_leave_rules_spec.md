# 特休假與請假管理規格書 (Annual Leave Interface Spec)

> **版本**: v1.1
> **狀態**: 草稿 (Draft)
> **目標**: 定義員工請假介面的互動邏輯與管理端的特休假設定介面。

---

## 1. 頁面功能概觀與角色權限

本模組涵蓋兩個主要介面：
1. **員工端 - 請假申請頁面 (`/app/leaves`)**：允許員工檢視特休餘額，並填寫包含多日拆分的請假單。
2. **管理端 - 員工資料設定 (`/app/admin/employees/[id]`)**：允許管理員設定員工的特休計算基準（到職日）與手動調整餘額。

---

## 2. 介面佈局與資料流 (UI Layout & Data Flow)

### 2.1 員工請假介面 (`/app/leaves`)
- **特休餘額顯示**: 在申請表單上方，明顯顯示「特休剩餘天數」。
    - 公式：`users.annual_leave_total - users.annual_leave_used`。
- **日期選擇與每日明細 (Daily Breakdown)**:
    - 使用者選擇「開始日期」與「結束日期」後，系統應自動展開列出該區間內的「每一天」。
    - **狀態切換**: 每一天皆可獨立設定狀態：
        - **全天 (1.0)**: 預設值。
        - **半天 (0.5)**: 適用於上午或下午請假。
        - **不請假 (0)**: 適用於週間休息日或國定假日。
    - **視覺回饋**:
        - 設定為「不請假」的日期，應以灰色背景及刪除線標示。
        - **例假日 (週六、週日)**: 應以 **Rose/Red** 色系區隔 (e.g., 背景色 `bg-rose-50`, 文字 `text-rose-600`) 以利識別。
- **自動計算 (Auto-Calculation)**:
    - 系統依據每一日的狀態自動加總產生「請假總天數」。
    - **防呆**: 禁止使用者手動輸入總天數。

### 2.2 管理端設定介面 (`/app/admin/employees/[id]`)
- **到職日設定**:
    - 新增 `DatePicker` 用於設定 `onboard_date`。
    - 變動後應提示管理員「此變更將影響未來的特休排程計算」。
- **手動調整**:
    - 允許管理員手動修正 `annual_leave_total` 欄位，以因應特殊勞資協議或補償發放。

---

## 3. 特殊頁面邏輯與狀態管理 (Local Logic & State)

- **請假申請送出防呆**:
    - 申請特休時，若 `申請總天數 > 剩餘天數`，必須即時阻擋送出並在按鈕旁顯示警告。
    - 若 `申請總天數 <= 0`，代表選取區間內無有效請假日，阻擋送出。
- **狀態管理**:
    - 申請送出時，應採用樂觀更新先行寫入列表，或利用 `isLoading` 鎖定 `ActionDialogs` 之送出按鈕。
    - 成功後透過 `revalidatePath` 刷新伺服器元件，重新獲取最新的 `annual_leave_used` 以防超休。
