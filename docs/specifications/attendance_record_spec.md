# 考勤記錄頁面規格書 (Attendance Record Spec)

> **版本**: v1.2
> **狀態**: 已實作 (Implemented)
> **目標**: 定義「打卡記錄 (Attendance Record)」頁面之月曆檢視與資料編輯互動。

---

## 1. 頁面功能概觀與角色權限

本頁面提供日曆式的出勤記錄檢視，並允許使用者點擊進行修正（補登/修改）。
- **一般員工 (Employee)**: 僅能查看**自己**的出勤紀錄，無法切換員工篩選器。
- **管理層級 (Manager / Super Admin)**: 預設查看自己，可透過上方「員工下拉選單」切換並檢視任何員工的當月打卡細節與修訂歷程。

---

## 2. 介面佈局與資料流 (UI Layout & Data Flow)

### 2.1 頁面結構
- **操作列 (Toolbar)**:
    - 月份選擇器 (YYYY-MM)，並附帶左右快速切換上/下個月之按鈕。
    - 員工選擇器 (僅管理員可見，顯示格式：`姓名 員編`。若員工尚未設定員編，則僅顯示姓名，不得顯示空括號 `()`)。
- **日曆視圖 (Calendar Grid)**: 
    - 採用 RWD 排版。手機版為單列垂直堆疊，桌面版為七列月曆佈局 (`grid-cols-7`)。
        - **大螢幕日曆自動補位**: 桌面版需依照當月第一天為星期幾進行空白底板補位 (`padding-start`)，確保每週的七天能固定對齊該欄位（如週日在第一欄，週六在第七欄）。手機版則預設隱藏這些補位區塊，維持由上往下的緊湊條列。

### 2.2 日卡片 (Day Card) 視覺呈現規則
每個「日期」為一個獨立的卡片，背景色應區分今日（藍色系）與週末（紅色系）。依據員工的「薪資類型」會有不同的展示邏輯：

| 顯示項目 | 月薪制 (Monthly) | 鐘點制 (Hourly) |
| :--- | :--- | :--- |
| **打卡時間** | 顯示上下班時間 (HH:mm) | 顯示上下班時間 (HH:mm) |
| **異常狀態標籤** | **顯示** (遲到/早退 - 紅橘字) | **依設顯示** (受開關控制) |
| **午休時數** | **顯示** (若 > 0) | **顯示** (e.g., "休 1h") |
| **請假資訊** | 顯示 (中文假別黃標) | 顯示 (中文假別黃標) |
| **工作時數** | 不顯示於卡片表面 | 不顯示於卡片表面 |

## 3. 特殊頁面邏輯與狀態管理 (Local Logic & State)

點擊任何一格日卡片，將啟動下方的 **操作對話框 (`AttendanceActionDialog`)**，對話框會依照該日期的不同組合狀態，渲染出適合的分頁 (Tabs)：

### 3.1 狀態 1: 無任何記錄 (Empty)
- **觸發**: 該日沒有任何打卡紀錄與請假申請。
- **可執行動作**: 
  - [補登打卡]: 從無到有新增上下班時間。
  - [新增請假]: 開啟請假申請流程。

### 3.2 狀態 2: 已有打卡記錄 (Has Attendance)
- **觸發**: 該日已有打卡紀錄。
- **可執行動作**:
  - [修改打卡]: 變更原本填寫的時間，送出後該筆紀錄會被記上 `is_edited=true` 標記。
  - [補請假]: 將剩餘的未出勤時間（如下半天）送出假單。

### 3.3 狀態 3: 已有請假申請 (Has Leave)
- **觸發**: 該日已有任何狀態的請假紀錄。
- **可執行動作**:
  - [補登打卡]: 用於處理非全天請假（半天假），補足剩餘應出勤時間。
  - [取消請假]: 允許員工撤回 `Pending` 狀態之假單。若已核准，則顯示相關聯絡提示。

### 3.4 假別顯示轉換 (Localization)
資料庫的 `leave_type` 欄位為英文字元 (如 `sick` / `annual`)，前端讀取後**嚴禁**直接顯示原始值，必須透過內建的 `LEAVE_TYPE_MAP` 轉換為中文（如 `病假` / `特休`），並以顯眼的黃底標籤呈現。

### 3.5 單邊打卡修改支援 (Single-sided Modification)
- **情境**: 員工當日已經上班打卡，但**尚未下班**，卻需要由管理員先行修改其上班時間（例如補登忘記打卡的紀錄）。
- **表單規範**:
  - `AttendanceActionDialog` 中的「下班時間 (clock_out_time)」欄位為**非必填 (Optional)**。
  - 對於鐘點制人員，需提供「清除下班時間」的按鈕機制，允許將誤填的下班時間恢復為空值。
- **計算引擎防呆 (Server-side)**:
  - 當後端 (`updateAttendance`) 接收到修改請求且 `clockOutTime` 為 `null` 時。
  - 系統**僅重算遲到狀態 (Late)**。
  - 系統**必須跳過且不計算**「早退狀態 (Early Leave)」與「當日淨工時 (Work Hours)」。工時欄位將被寫入為 `null`，以便日後員工實際執行下班打卡時，系統能以最新的狀態再次結算。
